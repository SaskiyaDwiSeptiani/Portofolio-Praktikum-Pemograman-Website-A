## ğŸ§© Kesimpulan Workflow Git

Dari riwayat di atas, dapat disimpulkan bahwa:
1. Git digunakan secara bertahap, setiap perubahan disimpan menggunakan commit yang terpisah untuk memudahkan pelacakan versi.  
2. Branch digunakan untuk eksperimen desain (styling) agar perubahan tidak langsung memengaruhi branch utama.  
3. Merge dilakukan setelah eksperimen selesai, memastikan versi akhir sudah stabil dan siap diunggah ke GitHub.  
4. Repository terhubung ke GitHub (origin/main), yang memudahkan dokumentasi, kolaborasi, dan publikasi hasil kerja.
